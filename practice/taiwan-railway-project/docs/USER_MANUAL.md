## 系統概述

台鐵車站資訊查詢系統是一個基於 Python tkinter 的桌面應用程式，提供直觀的圖形化介面來查詢和分析台鐵車站資訊及進出站人數資料。

### 主要功能

- **車站搜尋**: 快速搜尋和查看車站基本資訊
- **客流量查詢**: 查詢特定車站的進出站人數統計
- **車站比較**: 比較多個車站的客流量數據
- **圖表視覺化**: 以圖表形式展示客流量趨勢
- **資料匯出**: 將查詢結果匯出為 CSV 檔案或圖片

### 適用對象

- 交通運輸研究人員
- 政府機關相關人員
- 學術研究者
- 對台鐵數據感興趣的一般使用者

## 系統需求

### 最低系統需求

- **作業系統**: Windows 10/11, macOS 10.14+, Ubuntu 18.04+
- **Python 版本**: Python 3.8 或更高版本
- **記憶體**: 至少 4GB RAM
- **硬碟空間**: 至少 500MB 可用空間
- **網路連線**: 需要連接到 PostgreSQL 資料庫

### 建議系統需求

- **記憶體**: 8GB RAM 或更多
- **硬碟空間**: 1GB 可用空間
- **顯示器解析度**: 1920x1080 或更高

## 安裝指南

### 步驟 1: 安裝 Python

1. 前往 [Python 官方網站](https://www.python.org/downloads/)
2. 下載適合您作業系統的 Python 3.8+ 版本
3. 執行安裝程式，確保勾選「Add Python to PATH」選項

### 步驟 2: 下載應用程式

1. 從專案儲存庫下載最新版本
2. 解壓縮到您選擇的目錄

### 步驟 3: 安裝相依套件

開啟終端機或命令提示字元，導航到應用程式目錄，執行：

```bash
pip install -r requirements.txt
```

### 步驟 4: 設定資料庫連線

1. 確保您有 PostgreSQL 資料庫的存取權限
2. 設定環境變數或修改 `config.py` 檔案中的資料庫連線設定：

```python
DATABASE_CONFIG = {
    'host': 'your_database_host',
    'port': '5432',
    'database': 'taiwan_railway',
    'user': 'your_username',
    'password': 'your_password',
}
```

### 步驟 5: 啟動應用程式

在應用程式目錄中執行：

```bash
python main.py
```

## 快速開始

### 首次啟動

1. 啟動應用程式後，系統會自動檢查資料庫連線
2. 如果連線成功，狀態列會顯示「資料庫已連線」
3. 如果連線失敗，請檢查資料庫設定

### 基本操作流程

1. **選擇功能分頁**: 點擊上方的分頁標籤選擇所需功能
2. **輸入查詢條件**: 根據不同功能輸入相應的查詢條件
3. **執行查詢**: 點擊查詢按鈕開始搜尋
4. **查看結果**: 在結果區域查看查詢結果
5. **匯出資料**: 如需要，可將結果匯出為檔案

## 功能說明

### 車站搜尋

車站搜尋功能讓您快速找到特定車站的詳細資訊。

#### 使用方法

1. 切換到「車站搜尋」分頁
2. 在搜尋框中輸入車站名稱或代碼
3. 系統會即時顯示匹配的車站清單
4. 點擊清單中的車站查看詳細資訊

#### 搜尋技巧

- 支援部分匹配搜尋（例如：輸入「台北」會找到所有包含「台北」的車站）
- 可以使用車站代碼進行精確搜尋
- 搜尋不區分大小寫

#### 顯示資訊

- 車站代碼
- 車站名稱
- 地址
- 電話號碼
- GPS 座標
- 是否提供自行車租借服務

### 客流量查詢

客流量查詢功能讓您查看特定車站在指定時間範圍內的進出站人數統計。

#### 使用方法

1. 切換到「客流量查詢」分頁
2. 從下拉選單選擇要查詢的車站
3. 設定查詢的日期範圍（起始日期和結束日期）
4. 點擊「查詢」按鈕
5. 查看結果表格和統計摘要

#### 查詢選項

- **車站選擇**: 從所有可用車站中選擇
- **日期範圍**: 設定查詢的時間區間
- **資料排序**: 可按日期、進站人數或出站人數排序

#### 統計資訊

- 總進站人數
- 總出站人數
- 平均每日進站人數
- 平均每日出站人數
- 最高單日進站人數
- 最高單日出站人數

### 車站比較

車站比較功能讓您同時比較多個車站的客流量數據。

#### 使用方法

1. 切換到「車站比較」分頁
2. 點擊「新增車站」按鈕選擇要比較的車站（最多5個）
3. 設定比較的日期範圍
4. 點擊「開始比較」按鈕
5. 查看比較結果和排名

#### 比較指標

- 總客流量排名
- 進站人數排名
- 出站人數排名
- 平均每日客流量

#### 限制說明

- 最多只能同時比較 5 個車站
- 所有車站使用相同的日期範圍
- 如果某車站在指定期間無資料，會在結果中標註

### 圖表視覺化

圖表視覺化功能提供客流量數據的圖形化展示。

#### 圖表類型

- **趨勢線圖**: 顯示客流量隨時間的變化趨勢
- **長條圖**: 比較不同時期的客流量
- **堆疊圖**: 同時顯示進站和出站人數

#### 使用方法

1. 切換到「圖表視覺化」分頁
2. 選擇車站和日期範圍
3. 選擇圖表類型
4. 點擊「生成圖表」按鈕
5. 使用圖表工具進行縮放、平移等操作

#### 圖表功能

- **縮放**: 滑鼠滾輪或縮放工具
- **平移**: 拖拽圖表
- **重置**: 恢復原始視圖
- **匯出**: 儲存圖表為圖片檔案

### 資料匯出

所有查詢結果都可以匯出為外部檔案。

#### 匯出格式

- **CSV 檔案**: 適合在 Excel 或其他試算表軟體中開啟
- **PNG 圖片**: 適合插入文件或簡報
- **SVG 向量圖**: 適合需要高品質圖形的場合

#### 匯出方法

1. 完成查詢後，點擊「匯出」按鈕
2. 選擇匯出格式和儲存位置
3. 確認匯出設定
4. 等待匯出完成

## 常見問題

### Q: 應用程式啟動時顯示「資料庫連線失敗」怎麼辦？

A: 請檢查以下項目：
- 確認資料庫伺服器正在運行
- 檢查網路連線
- 驗證資料庫連線設定（主機、埠號、使用者名稱、密碼）
- 確認資料庫使用者有適當的存取權限

### Q: 搜尋結果為空是什麼原因？

A: 可能的原因包括：
- 搜尋條件過於嚴格
- 資料庫中沒有匹配的資料
- 日期範圍設定錯誤
- 車站代碼輸入錯誤

### Q: 圖表顯示不正常怎麼辦？

A: 請嘗試以下解決方法：
- 重新整理資料
- 調整日期範圍
- 檢查資料是否充足
- 重新啟動應用程式

### Q: 匯出功能無法使用？

A: 請確認：
- 有足夠的硬碟空間
- 對目標資料夾有寫入權限
- 檔案名稱不包含特殊字元
- 沒有其他程式正在使用同名檔案

### Q: 應用程式運行緩慢？

A: 可以嘗試：
- 縮小查詢的日期範圍
- 清除應用程式快取
- 關閉其他不必要的程式
- 檢查網路連線速度

## 故障排除

### 記憶體使用過高

如果應用程式使用過多記憶體：

1. 點擊選單「工具」→「效能管理」→「記憶體使用狀況」
2. 查看記憶體使用詳情
3. 點擊「清理記憶體」按鈕
4. 如果問題持續，請重新啟動應用程式

### 資料載入緩慢

如果資料載入速度過慢：

1. 檢查網路連線速度
2. 嘗試縮小查詢範圍
3. 清除快取：選單「檢視」→「清除快取」
4. 檢查資料庫伺服器負載

### 圖表渲染問題

如果圖表無法正常顯示：

1. 確認已安裝 matplotlib 套件
2. 更新圖形驅動程式
3. 嘗試不同的圖表類型
4. 重新啟動應用程式

### 匯出失敗

如果資料匯出失敗：

1. 檢查目標資料夾的寫入權限
2. 確認有足夠的硬碟空間
3. 檢查檔案名稱是否包含非法字元
4. 嘗試匯出到不同位置
